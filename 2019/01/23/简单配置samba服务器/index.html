简单配置samba服务器

发表于 2019-01-23 | 更新于: 2019-01-23 | 分类于 Linux | 评论数 | 热度: ℃

samba是实现Linux和Windows计算机之间相互通信的一个免费软件，核心是使用SMB（Server Message Block）协议。SMB协议工作于会话层、表示层和一小部分应用层，它使用了NetBIOS的应用程序接口。

运行环境：

  • 虚拟机：VMware® Workstation 14 Pro
  • 服务器：CentOS 7 64位
  • 客户端：Windows 8

Share安全级别服务器

任务：建立测试目录和测试文件，并将测试目录共享，不需要用户名和密码即可访问。

安装samba

打开终端，输入 yum install samba -y 进行安装。如果安装不成功，请检查yum源配置，网络等相关内容。

建立用于共享的测试目录和文件

打开 test.txt 文件，并在里面填入内容。

修改主配置文件

打开samba的主配置文件，位于 /etc/samba/smb.conf ，进行修改。

全局变量区域配置，由于samba4开始不支持share安全级别， security 如果设置为 share ，会导致服务无法启动，所以这里设置为 user，并加上一项配置内容 map to guest = Bad User 实现不需要输入用户名密码即可访问共享目录的目的。

共享区域配置

启动samba服务

使用命令 systemctl start smb 对服务进行启动，如果已经启动，则需要使用 systemctl restart smb 重启。

启动后使用命令 systemctl status smb 查看samba运行状态。

运行状态显示 active (running) 即为活动状态，运行成功。

验证

打开客户机（windows 8）的资源管理器，在地址栏输入服务器IP地址 \\192.168.15.137 ，回车。如图所示，可以访问共享的目录和文件，并且不需要身份验证。

user安全级别服务器

任务：将测试目录共享，并且指定用户才可访问。

安装samba，建立测试目录和文件的两步见上文。

创建用户

用于客户端访问samba服务器时进行身份验证的用户名称为samba帐号。samba账号不能直接被创建，必须存在对应的同名系统账号，故如果要创建一个新的samba账号，则必须先创建同名的系统账号。

创建系统账号

创建用户组“group1”，并为该组内添加用户“user1”。创建非“group1”组内用户user2。

注：为了安全，在输入密码时，并不会有任何显示信息，为正常现象。

创建samba账号

由于已经创建了系统账号，所以可以直接创建同名的samba账号。参数 -a 表示新添加的samba帐号，如果只是修改samba账号密码，去掉该参数即可。

修改主配置文件

打开samba的主配置文件，位于 /etc/samba/smb.conf ，进行修改。

全局变量区域配置

共享区域配置

可以根据需求添加相应字段，其中 path 字段为必需。 browseable 字段设置能否在客户端显示，如果值为 no 则会隐藏。valid users 字段设置允许访问该共享目录的用户或用户组，值为用户名，或用户组名，其中用户组名前需要加上 @ 符号，多个用户/用户组逗号隔开，例：validusers = user1,user2,@group1。writable 字段设置该目录是否可写。

设置本地系统权限

由于设置共享区域的 writable 字段值为 yes ，表示对该目录拥有写权限，为了保证写操作的正常执行，需要设置该目录的系统权限。

将共享目录设置为最大权限777。

重启samba服务

使用命令 systemctl restart smb 重启服务。

验证

打开客户机（windows 8）的资源管理器，在地址栏输入服务器IP地址 \\192.168.15.137 ，回车。如图所示，要求输入帐户密码才可以访问。

当输入user1帐户和密码时，可以访问该共享目录，并且里面的文件也可以查看。

当输入user2帐户和密码时，并不可以访问该共享目录，提示拒绝访问。

user1可以访问，user2不可以访问，原因是在主配置文件中，我们设置了 valid users = @group1。规定只有属于用户组 group1 内的用户才可以访问，而在创建用户时指定，user1是属于group1组的，user2不属于，故user2不可访问。

故障排除

无法找到 192.168.15.137

如果在客户端（windows 8）验证时，弹出无法找到对话框，则按照以下方法逐个排除故障。

网络

使用ping命令对网络的连通性进行测试，确保客户端和服务器的网络可以连通。

客户端

在windows 8中使用ping命令的方法：按下组合键 win+r ，打开“运行”对话框，输入cmd，点击确定，打开命令提示符。

输入 ping 192.168.15.137 ，此处的IP地址为服务器的IP。

服务器

在Linux中，打开终端输入 ping 192.168.0.104 -c 4 ，此处的IP地址为客户端的IP，-c表示需要回应次数，4为4次，如果不加-c参数，则会无限发送数据包，按下 ctrl+c 组合键可以终止。

如果不能ping通，则为网络故障。

考虑是否是虚拟机网络设置错误，选择虚拟机，右键打开设置对话框，检查网络适配器的设置是否有误，我这里采用NAT方式映射主机IP。

防火墙

服务器端的防火墙需要放行samba服务，否则会拦截samba数据包，造成无法找到IP的错误。

两种解决办法：

  • 关闭防火墙
  • 将samba服务添加至防火墙信任列表

关闭防火墙

此方法简单有效，如果没有使用防火墙的需要，可以直接关闭，终端中输入命令 systemctl stop firewalld

配置防火墙信任列表

在菜单中找到“应用程序→杂项→防火墙”，打开“防火墙配置”对话框。在“配置”下拉列表中选择“永久”，然后切换到“区域”选项卡，在左侧的“区域”列表中选中“public”选项，在右侧切换到“服务”选项卡，在此选项卡中勾选“samba”和“samba-client”两个选项。

SELinux

由于SELinux会影响samba共享目录的使用，所以需要将SELinux关闭。

两种解决办法：

  • 临时关闭，在终端输入命令 setenforce 0，该方法重启系统后失效。
  • 修改配置文件进行永久性关闭。

打开SELinux的配置文件，位于 /etc/selinux/config ，进行修改。

将 SELINUX 字段的值修改为 disabled ，保存，关闭，重启系统后生效。

windows配置

如果服务器和客户端可以ping通，网络正常，并且服务器的防火墙和SELinux都没有问题，还提示无法找到的问题，考虑是否为客户端个别服务没有启动的问题。

解决方法：
按下 win+r 组合键打开“运行”对话框，输入 services.msc 后单击确定，打开“服务”窗口，检查下列服务是否正在运行。

  • Server
  • TCP/IP NetBIOS Helper
  • Workstation

如果服务没有运行，右键“属性”，修改“启动类型”为 自动 ，单击“应用”，生效后单击“启动”，服务状态显示“正在运行”即可。
