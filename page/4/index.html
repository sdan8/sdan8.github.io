四种XML解析方式比较

发表于 2016-12-04 | 更新于: 2016-12-04 | 分类于 XML | 评论数 | 热度: ℃

优缺点比较

  • DOM：基础方法，平台无关的官方解析方式
  • SAX：基础方法，基于事件驱动的解析方式

  • JDOM、DOM4j：在基础的方法上扩展出的，只有在java中能够使用的解析方法

  • DOM：一次性将整个XML文件加载入内存，如果文件较大则对性能要求高

  • SAX：基于事件驱动，会根据XML文件内容逐条解析
    • <?xml?>标签触发startDocument()方法
    • 每个开始标签触发startElement()方法
    • 每个结束标签触发endElement()方法
    • 到根节点的结束标签后触发endDocument()方法

RandomAccessFile

发表于 2016-12-04 | 更新于: 2016-12-04 | 分类于 Java | 评论数 | 热度: ℃

java提供的对文件内容的访问，既可以读文件，也可以写文件。支持随机访问文件，可以访问文件的任意位置

步骤

  1. java文件模型
    • 在硬盘上的文件是 byte byte byte 存储的，是数据的集合
  2. 打开文件
    • 有两种模式 “rw”（读写） “r”（只读）
    • RandomAccessFile raf = new RandowAccessFile(file,”rw”)
    • 文件指针，打开文件时指针在开头 pointer = 0;
  3. 写方法
    • raf.write(int)–>只写一个字节（后8位），同时指针指向下一个位置，准备再次写入
  4. 读方法
    • int b = raf.read()–>读一个字节
  5. 文件读写完成以后一定要关闭（Oracle官方说明）
